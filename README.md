# Issues Study Lab

A web workspace for SACE Stage 2 Philosophy students working on the Issues
Study (AT3 Investigation). Five steps: read what SACE actually asks, pick a
question from a curated bank, browse the dialectic and reading list, talk
to the **Chamber** (a Socratic interlocutor that has read the cluster's
curated corpus), then draft a response and get rubric-referenced feedback.

## Stack (May 2026 — post-Vertex)

- **Frontend**: vanilla HTML/CSS/JS, hash-routed wizard, served from
  Netlify CDN.
- **Static data**: `data/questions.json`, `data/readings.json`,
  `data/rubric.json` — built from `pack_metadata.json` /
  `cache_handles.json` / `extracted_docx/` via `build_static_data.py`.
- **AI backend**: two Netlify Functions in `netlify/functions/`:
  - `chat.js` → `POST /api/chat` — Anthropic Haiku 4.5 with the cluster
    corpus cached at the system-prompt boundary (5-min ephemeral cache,
    auto-renews while the class is active).
  - `feedback.js` → `POST /api/feedback` — single-shot rubric grading,
    no caching.
- **Persistence**: per-question chat / draft / feedback in `localStorage`.
- **PDF export**: client-side `window.print()` against a print-only view
  injected at click time. No server round-trip.

## Cost design

The Anthropic API key has a hard $20/month cap. Function code is tuned
for that ceiling:

- `claude-haiku-4-5` ($1/MTok in, $5/MTok out, ~$0.10/MTok cached read).
- Chat output capped at 800 tokens; feedback at 1500.
- Last 12 chat turns kept; older history trimmed.
- Per-IP soft rate limit (30 req/hour per warm container).
- Token usage + estimated USD logged on every call.

Realistic bill for a 14-student / 5-week cohort: ~$30–50 in API spend.
The cap will catch any runaway.

## Reading content

The 14 cluster packs that the Chamber reads from live in `data/packs/`,
one `.txt` per cluster, generated by `build_packs.py` from:

- `readings/` — 98 small PDFs committed to the repo.
- `text_packs/` — pre-extracted plain text for 39 large readings (the
  ones that wouldn't fit inline in the previous Vertex caches).

`readings_manifest.txt` lists what to push.

## Running locally

```sh
npm install
echo "ANTHROPIC_API_KEY=sk-ant-..." > .env       # gitignored
npx netlify dev                                  # serves the site + Functions
```

Then open <http://localhost:8888>.

The first Chamber message after a quiet period writes the cache (~5–15 s
on a 50K-token cluster); subsequent messages within 5 min hit the cache
(~1–3 s).

## Deploying

This branch deploys to Netlify directly. One env var to set in the
Netlify dashboard before the first deploy:

| Site settings → Environment variables | Value |
|---|---|
| `ANTHROPIC_API_KEY` | your Anthropic API key |

Then push to a branch and Netlify auto-builds. No Docker, no Cloud Run,
no Vertex.

## Build steps (regenerating the static data)

```sh
python build_static_data.py     # → data/{questions,readings,rubric}.json
python build_packs.py           # → data/packs/<cluster>.txt
```

Both scripts are idempotent. Re-run after editing `pack_metadata.json` or
adding readings.

## Things that are gone

- `server.py` — the Flask broker (Cloud Run / Vertex).
- `Dockerfile`, `.dockerignore`, `.github/workflows/deploy.yml`.
- `cache_handles.json` — Vertex cache references; the Anthropic side
  caches automatically.
- The `/api/*` proxy in `netlify.toml` to Cloud Run; `/api/*` now
  rewrites to Netlify Functions.
- The `/readings/*` proxy to a GCS bucket; readings are committed and
  served from Netlify's CDN.
